Livario UI & UX Guidelines

Overview

Livario’s interface should feel:

- premium
- modern
- elegant
- calm
- trustworthy
- mobile-first

The experience should combine:

- Apple-level simplicity
- Airbnb-style usability
- Linear-style polish
- modern SaaS responsiveness

Livario is NOT a cluttered classified website.

It should feel like a premium digital product.

---

Core Design Philosophy

Minimalism First

Every screen should feel:

- spacious
- intentional
- easy to scan
- visually balanced

Avoid:

- overcrowded layouts
- excessive colors
- visual noise
- unnecessary borders
- dashboard clutter

---

User Experience Philosophy

The UX should prioritize:

- clarity
- speed
- simplicity
- discoverability
- trust

Users should always understand:

- where they are
- what actions are available
- what happens next

---

Design Inspirations

Primary inspirations:

- Apple
- Airbnb
- Linear
- Notion
- Arc Browser

---

Visual Identity

Tone

Livario should feel:

- refined
- modern
- confident
- lightweight
- premium

---

Color Philosophy

Use Colors Intentionally

Prefer:

- neutral foundations
- restrained accent colors
- soft contrast
- subtle depth

Avoid:

- overly saturated palettes
- random color usage
- excessive gradients

---

Suggested Palette Direction

Base

- whites
- soft grays
- charcoal tones

Accent

Use one consistent accent color throughout the product.

Accent usage should remain restrained.

---

Typography Standards

Typography is one of the most important parts of Livario’s identity.

---

Typography Principles

Text should feel:

- highly readable
- modern
- spacious
- balanced

Avoid:

- tiny unreadable text
- inconsistent weights
- excessive font sizes

---

Font Style Direction

Prefer:

- clean sans-serif typography
- modern spacing
- medium weight headings
- readable line heights

---

Spacing Philosophy

Spacing creates the premium feel.

Use:

- generous whitespace
- consistent spacing scales
- balanced layouts

Avoid:

- cramped UI
- inconsistent padding
- visually dense sections

---

Border Radius Philosophy

Use soft modern rounding consistently.

Suggested direction:

- cards: medium/large radius
- buttons: medium radius
- modals: large radius

Avoid:

- overly sharp corners
- inconsistent rounding

---

Shadow Philosophy

Shadows should be:

- subtle
- soft
- minimal

Avoid:

- harsh shadows
- excessive glow effects
- layered shadow chaos

---

Layout Philosophy

Mobile-First

Mobile is the primary experience.

All interfaces should:

- feel natural on mobile
- remain touch-friendly
- scale upward cleanly to desktop

---

Responsive Design Rules

Mobile

Prioritize:

- thumb reachability
- touch spacing
- simple navigation
- readable content

---

Tablet

Adapt layouts gradually.

---

Desktop

Use:

- wider grids
- sidebar layouts where appropriate
- improved content spacing

Avoid making desktop feel bloated.

---

Navigation Guidelines

Mobile Navigation

Use:

- bottom navigation where appropriate
- clean mobile menus
- minimal top navigation clutter

Onboarding should feel calm and decisive.

The first onboarding screen after signup should ask:

"How would you like to use Livario?"

It should present two clear actions:

- Continue as Tenant
- Continue as Landlord

Include the note:

"You can switch this later in Settings"

The experience should be minimal, centered, mobile-first, and avoid explaining future role complexity.

---

Desktop Navigation

Prefer:

- clean top navigation
- contextual sidebars
- minimal complexity

---

Component Philosophy

All UI should use reusable components.

Avoid:

- duplicated UI patterns
- inconsistent styles
- one-off implementations

---

Core Components

Buttons

Buttons should feel:

- clean
- responsive
- modern
- accessible

Button Rules

- consistent heights
- consistent padding
- clear hover states
- loading states

---

Inputs

Inputs should feel:

- spacious
- simple
- readable

Input Rules

- clear labels
- visible focus states
- accessible spacing
- validation feedback

---

Cards

Cards are central to Livario.

Used for:

- property listings
- dashboards
- user content
- summaries

Card Rules

- soft corners
- subtle borders/shadows
- clean spacing
- visual hierarchy

---

Property Card Philosophy

Property cards are one of Livario’s most important UI elements.

They should:

- emphasize imagery
- maintain clean typography
- feel premium
- remain highly scannable

---

Property Cards Must Prioritize

1. image quality
2. pricing visibility
3. property title
4. location clarity
5. clean metadata hierarchy

---

Imagery Standards

Property imagery is critical.

Images should:

- load quickly
- remain responsive
- maintain aspect ratio consistency
- feel immersive

Avoid:

- distorted images
- inconsistent image sizing
- cluttered overlays

---

Loading Experience

Loading states should feel polished.

Use:

- skeleton loaders
- smooth transitions
- optimistic UI where appropriate

Avoid:

- flashing layouts
- blank screens
- jarring loading behavior

---

Empty States

Empty states should:

- guide users
- feel intentional
- encourage action

Avoid dead-end screens.

---

Animation Philosophy

Animations should:

- feel subtle
- improve clarity
- improve responsiveness

Avoid:

- flashy animations
- excessive motion
- distracting transitions

---

Motion Guidelines

Prefer:

- subtle fades
- smooth transitions
- lightweight hover effects

---

Accessibility Standards

Accessibility is mandatory.

Maintain:

- keyboard accessibility
- semantic HTML
- readable contrast
- focus visibility
- accessible labels

---

Form Design Standards

Forms should:

- feel lightweight
- reduce cognitive load
- remain easy on mobile

Use:

- progressive disclosure
- grouped inputs
- clear validation

---

Dashboard Philosophy

Dashboards should feel:

- calm
- organized
- highly readable

Avoid:

- information overload
- excessive widgets
- cluttered analytics

---

Search Experience Philosophy

Search should feel:

- fast
- simple
- intuitive

Filters should remain:

- understandable
- collapsible on mobile
- visually lightweight

---

Modal & Dialog Philosophy

Modals should:

- remain focused
- avoid overwhelming users
- support mobile properly

Avoid giant complex modal systems.

---

Iconography

Use:

- simple outline icons
- consistent sizing
- restrained icon usage

Avoid decorative icon overload.

---

Notification Philosophy

Notifications should:

- remain concise
- provide clarity
- avoid interrupting workflows

---

Error Handling UX

Errors should:

- feel human
- remain calm
- explain next steps clearly

Avoid:

- technical jargon
- aggressive warning styles

---

Performance UX Standards

The UI should feel:

- instant
- lightweight
- responsive

Prioritize:

- fast interactions
- optimized rendering
- smooth navigation

---

Brand Experience

Livario should feel like:

- a premium startup product
- a trustworthy housing platform
- a modern mobile-native experience

not:

- an outdated classified marketplace

---

Consistency Rules

Maintain consistency across:

- spacing
- typography
- colors
- interactions
- navigation
- button behavior
- form styling

Consistency creates trust.

---

Engineering Collaboration Rules

Developers should:

- use reusable components
- avoid inline style chaos
- maintain design consistency
- follow spacing systems
- avoid one-off visual solutions

---

Final Principle

Every UI decision should answer:

“Does this feel premium, modern, and calm?”

Livario’s UI should make users feel:

- comfortable
- confident
- focused
- impressed

without feeling visually overwhelming.
